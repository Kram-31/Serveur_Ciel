# Fiche Révision : Projet Serveur Ciel (Mode Cyber)

## 1. L'Architecture
Un serveur centralisé sous Docker, protégé par un Reverse Proxy (Nginx) et une gestion DNS locale (AdGuard).

- **Entrée unique** : Port 80 (Nginx).
- **Sécurité** : Ports des services masqués (accessibles uniquement via sous-domaines).
- **Administration** : Gestion via Portainer (Web UI) au lieu de la ligne de commande.

## 2. Les Adresses (À connaître)

| Service | Adresse "Cyber" | Adresse Secours (IP) | Rôle |
| :--- | :--- | :--- | :--- |
| **Heimdall** | `http://ciel.lan` | `:8085` | Portail d'accueil |
| **FileBrowser** | `http://files.ciel.lan` | `:8080` | Serveur de fichiers |
| **AdGuard** | `http://adguard.ciel.lan` | `:3000` | DNS & Pare-feu |
| **Portainer** | `http://admin.ciel.lan` | `:9000` | Gestion Docker |
| **Glances** | `http://monitor.ciel.lan` | `:61208` | Monitoring (Ressources) |
| **CyberChef** | `http://cyber.ciel.lan` | `:8082` | Outils Crypto/Décodage |
| **IT-Tools** | `http://tools.ciel.lan` | `:8081` | Outils Réseau |

## 3. Identifiants
- **FileBrowser** : `admin` / `adminadminadmin`
- **Portainer** : Créé au lancement (`admin` / 12 caractères).
- **AdGuard** : Créé au lancement.

## 4. Procédure de Secours (Si ça plante)
1. Vérifier si les conteneurs tournent : `docker ps`
2. Vérifier si Nginx a planté : `docker logs nginx`
3. Si le DNS ne marche pas, utiliser les adresses de secours (IP + Port).

## 5. Le "Petit Plus" pour le Prof
> "J'ai mis en place une segmentation par sous-domaines pour simuler une architecture d'entreprise réelle, où chaque service est isolé derrière un proxy inverse pour des raisons de sécurité et de facilité d'audit."
